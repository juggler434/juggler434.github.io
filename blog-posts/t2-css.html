CSS Inline vs. Inline Block

Posted September 5, 2014

In this post I'm going to be taking a closer look at the difference between display: inline and display: inline-block in CSS. I have to admit, this is a topic that I struggled with in CSS, so I'm hoping this will bring clarification to others

I'm going to start with how inline and inline-block display differ from block display. The answer is very simple and right in the name. A block element forces a line break after it, while inline and inline-block allow elemnts to be laid next to each other.

So we've figured out we want to lay out our objects next to each other, now we have to figure out whether we want to use an inline display or an inline-block display. The basic difference is that the dimensions of an inline elemnt are set by its content, where inline-block elements can be adjusted just like blocks. This response on stack overflow does a pretty good job of summing up the difference. To summarize inline elements will respect left and right padding and margins, but not top and bottom, and cannot have a set height and width. Alternatively, inline-block respects margin and padding on all sides, and can have a set height and width.

To see and example of an inline element, you have to look no further than the link above. <a> tags default to inline, which allows it rest nicely within the paragraph.
